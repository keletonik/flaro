OVERVIEW
────────────────────────────────────────────────────────────────
Build AIDE — a premium personal operations assistant web and 
mobile app. It functions as an intelligent PA that the user 
talks to conversationally, drops emails into, and uses to 
manage jobs, notes, and actions — all from one interface.

The app combines a command centre dashboard (at-a-glance 
status of everything) with a conversational AI chat interface 
(the primary interaction method). Everything the user types 
or drops into the chat is automatically categorised, logged, 
and surfaced in the dashboard.

The user's name is Casper. The AI should always address him 
by name and behave as a senior personal assistant who knows 
his business context deeply.

DATABASE STRUCTURE
════════════════════════════════════════════════════════════════

Replit KV keys:

  chat_history          → array of message objects (last 100)
  jobs_index            → array of job IDs
  job_[id]              → individual job JSON
  notes_index           → array of note IDs
  note_[id]             → individual note JSON
  toolbox_index         → array of toolbox IDs
  toolbox_[id]          → individual toolbox note JSON
  oncall_roster         → weekly roster JSON (manual entry)
  settings              → app config JSON

JOB OBJECT SCHEMA:
{
  id: string,
  task_number: string,
  site: string,
  address: string,
  client: string,
  contact_name: string,
  contact_number: string,
  contact_email: string,
  action_required: string,
  priority: 'Critical' | 'High' | 'Medium' | 'Low',
  status: 'Open' | 'In Progress' | 'Booked' | 'Blocked' | 
          'Waiting' | 'Done',
  assigned_tech: string,
  due_date: string,
  notes: string,
  uptick_notes: array of strings,
  created_at: timestamp,
  updated_at: timestamp
}

NOTE OBJECT SCHEMA:
{
  id: string,
  text: string,
  category: string,
  owner: string,
  status: 'Open' | 'Done',
  created_at: timestamp
}

CRITICAL BUILD RULES
════════════════════════════════════════════════════════════════

1.  Anthropic API key stored as Replit Secret: ANTHROPIC_API_KEY
    Never hardcoded anywhere in the codebase.

2.  All AI calls routed through the Express backend.
    Frontend never calls Claude directly.

3.  All errors show user-friendly messages. Never raw errors.

4.  All destructive actions require confirmation.

5.  All saves show a green toast: "Saved" (bottom, 2.5s).

6.  App must be fully responsive:
    Mobile: bottom tab navigation
    Web/tablet: left sidebar navigation (240px wide)

7.  On first load with empty database:
    Show welcome screen with AIDE intro message in chat:
    "Hi Casper — I'm AIDE, your personal operations assistant. 
    Drop an email in here, give me a note to log, or ask me 
    anything about your jobs. I'll handle the rest."

8.  Use claude-sonnet-4-20250514 for all AI.
    Temperature: 0.3 for structured outputs (triage, jobs)
    Temperature: 0.6 for conversational responses
    Max tokens: 2000

9.  The chat is the PRIMARY interface. 
    Everything else is a view of what the chat has logged.
    Design around this — chat is the hero, dashboard is 
    the reference layer.

10. Build for Expo Go compatibility so Casper can test 
    on his phone immediately by scanning a QR code.

11. The app name throughout is AIDE — no FlameSafe branding 
    in the app itself (this is a Mentaris product).

12. Use Expo Router for navigation. 
    app/(tabs)/index.tsx      → Dashboard
    app/(tabs)/chat.tsx       → AIDE Chat
    app/(tabs)/jobs.tsx       → Jobs
    app/(tabs)/notes.tsx      → Notes
    app/(tabs)/toolbox.tsx    → Toolbox
    app/job/[id].tsx          → Job Detail
