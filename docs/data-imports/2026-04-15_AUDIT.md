## Step 3 — Import

**Method:** pre-parsed the CSVs offline via `scripts/build-apr15-seed-json.cjs` into a single JSON blob at `artifacts/api-server/src/seed-apr15-batch.json`, then wired a boot-time loader at `artifacts/api-server/src/seed-apr15-batch.ts` that runs after every existing seed on Replit restart.

**Post-dedup row counts** (unique records after collapsing duplicates across the 3 task exports + 2 quote exports + 2 cycle-time exports):

| Collection | Rows |
|---|---:|
| `wip_records` | 264 |
| `quotes` | 96 |
| `defects` | 204 |
| `task_cycle_times` | 9,743 |
| `email_notes` (→ `notes` with `category='Follow Up'` + `raw_data.kind='email-triage'`) | 46 |

**Idempotency:** every insert path checks existence by natural key first. `wip_records` and `quotes` patch the mutable hot columns (status, priority, assigned_tech, quote_amount) on re-run; `task_cycle_times` patches status + cycle columns. `defects`, `notes` are insert-or-skip. The loader short-circuits completely if the batch id already has the full row count in `wip_records`.

**Never-delete rule:** no `UPDATE … SET deleted_at = …` anywhere, no `DELETE` anywhere, no `TRUNCATE`. Every existing row is preserved bit-for-bit.

## Step 5 — Metrics

Two new metric definitions added to the registry, both reading from the new `task_cycle_times` table:

| Metric | Category | Unit | What it shows |
|---|---|---|---|
| `task_cycle_time_distribution` | ops | days | Tasks bucketed by days-to-complete (0-1 / 2-5 / 6-14 / 15-30 / 31-60 / 61-120 / 120+). Headline is median days-to-complete in the window. |
| `invoice_lag_distribution` | revenue | days | Tasks bucketed by days-to-invoice (same day / 1-3 / 4-7 / 8-14 / 15-30 / 31+). Headline is mean billing lag in the window. |

Both are queryable at `GET /api/metrics/:id?period=90d` and via the agent's `metric_get` tool immediately. No other metrics needed changes — the existing 10 already read from `wip_records` / `quotes` / `defects` / `invoices`, which all got fresh data from the import.

**Registry totals:** 12 metrics (up from 10).
